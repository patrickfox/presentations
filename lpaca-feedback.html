Best Practices

Use of aria-expanded

  • aria-expanded is for use on interactive elements only (button, a)
Example

Use of aria-label

aria-label has a tendency to override and speak for any inner text

  • Currently used on all "Apply Now" CTA's
  • In most cases, the "Apply Now" text is not read
Example

Use of landmarks

  • LPACA pages are missing landmarks
  • role='main' and role='contentinfo' should demarcate main and footer content areas
  • Normal pages with distinct header and navigation would provide these landmarks as well
Example

Anchors as buttons üëé

  • Use button for functionality and form submit
  • Use a for true links (a link to a new page/view)
  • Avoid role='button' and role='link'
  • Use HTML that describes the functionality, not what something looks like

Text Color Contrast Ratio üåà

The comps provide a style guide that documents and prescribes valid, color contrast compliant text-to-background color combinations. This guide provides color hex values and contrast ratios, and serves as a usage guide for developers and a confirmation of accessibility compliance for QA.

Examples:

  • Text superimposed over dynamic image or video
  • Requirement: text-to-background color contrast ratio for all text must be at least 4.5:1.
  • One exception - a lighter 3:1 contrast ratio will suffice if:
    • the text size is at least 14pt/19px and bold, or
    • the text is 18pt/24px and normal weight

Process Updates

Tech

  • Before delivery to QA, check accessibility at each break point using Chrome Lighthouse A11y Audit
  • Employ best practices outlined above
