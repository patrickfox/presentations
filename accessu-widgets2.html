File Selector and Upload

Overview

  • Modern file upload supports multiple uploads and provides progress bars and status
  • Goal: communicate dynamic updates to file upload status in an accessible manner

File Upload Example

Experience Design

Provide accessible notifications for each status update

  • Visible notifications update dynamically
  • announce() important notifications for AT users

Experience Design
Requirements

Ensure that upload instructions and requirements are available

  • "Maximum file size is 10 megabytes"

Experience Design
On Upload

  • "Uploading 5 files to 'My Photos'"

Experience Design
On File/Upload Complete

  • "Uploaded 3 of 5 files"
  • "3 of 5 files uploaded, 50% complete"
  • "5 of 5 files uploaded - upload complete"

Experience Design
On Error/Cancel

  • "Upload failed - 4 of 7 files were uploaded"
  • "Upload cancelled - 2 of 5 files not uploaded"

Experience Design
On Unload

  • Warn users - upload is not complete

Implementation

Status updates are made possible via XMLHttpRequest2 object's upload property

xhr.upload.onprogress = function(e) {
 if (e.lengthComputable) {
   progressBar.value = (e.loaded / e.total) * 100;
   var message = 'Files' + progressBar.value + '% loaded';
   $.announce(message, 'assertive');
 }
};

Example

  • Google Photos

Form Validation

Overview

Make success easy for all users, provide info and assistance

  • Field labels convey purpose
  • Requirements - required inputs, formatting, instructional info
  • Errors and help - what is wrong and how to fix it

Form Validation Example

Implementation

Implementation
Labels

Form fields and groups of related radios/checkboxes must be labelled

Techniques:

  • <label for="{id}">
  • aria-label="label copy"
  • aria-labelledby="{idref}"

Implementation
Requirements

Required fields and formatting or instructional copy must be associated with a field and understood via AT

  • Required field - use aria-required="true"
  • Formatting and instructional copy - use aria-describedby="{idref}"

Implementation
Errors and Help

Error states and messaging must be communicated via AT

  • Field error state - use aria-invalid="true"
  • Error text - use aria-describedby="{idref}" or "label error" technique
  • Additional copy - prepend "Error: " to error message
  • Onblur - announce errors politely

Implementation
Example

  • Sears Card - Sign On Form

Search By Location

Overview

Dynamic location search with map view

  • Maps OK when combined with text results
  • Search results displayed dynamically
  • Filtering options update search results
  • Make current search status available to all users

Store Locator Example

Experience Design

  • As results are updated/filtered, display and announce a notification
  • Ensure that filtered state and number of results is communicated
  • Ensure that filter controls are keyboard navigable

Experience Design
On Search/Filter Update

  • Dynamically update map and results
  • Communicate update and status
    • "Found 18 open stores near 78701"
    • "Showing locations in 55442, 2 filters active"

Implementation
Map and Search Results

  • Hide map from screen reader via aria-hidden
  • Ensure equivalent alternative to map

Implementation
On Search/Filter Update

  • Update search/filter status message
  • Manage focus
    • Place focus on visible summary (e.g. $.access())
    • Not visible? Hidden heading, button description

Examples

  • Starbucks - Store Locator

Improving Single Page Applications

Communicate View Changes

One approach: Emulate native browser page-load behavior

  • Page title is used to announce page load/refresh
  • Stating that page/view is "loaded" is helpful
Page load behaviors

Step 1: Update Page Title

  • Significant UI/context change requires a title change
  • <title> plays an important role in orientation
  • SPA frameworks don't automate title updates

Step 2: Announce view change

  • Live regions are great for announcing content unobtrusively
  • Create a message: {page_title} + " loaded"
  • Update live region → message is spoken by screen reader
