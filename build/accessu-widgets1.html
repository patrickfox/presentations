Focus Management

Focus Management

  • Care for user focus
  • Ensure that UI updates do not change user focus unless:
    • A focus change is intended and necessary
    • A focus change improves the user's understanding of a context change
    • Examples: modals, drop-down menus, content is no longer available in new context

Tabindex Usage

  • tabindex="0" makes an element keyboard focusable, uses natural tab order
  • tabindex="{≥1}" makes an element keyboard focusable, assigns specific order
  • tabindex="-1" makes an element programmatically focusable (e.g. via JS)

Tabindex Best Practices

  • Never use tabindex values > 0
  • Ad hoc use of tabindex can cause a lot of unintended issues
  • Avoid the use of tabindex - rely on natural tab order

Use Buttons and Links

Need a clickable/user-interactive element?

  • Use a button or a for intended purpose
    • Use a to link to another location/view
    • Use button for generic clickable or form submit

Use Buttons and Links

Antipatterns (do not do this):

  • div(role="button" tabindex="1") or div(role="button" tabindex="1")
  • a(role="button") * or button(role="link") *

* Except with remediation efforts

Focus On Anything

Need to put focus somewhere in the UI that isn't focusable?

  • Use a script to manage focus and place focus temporarily
  • No need to manually hard code tabindex in HTML
  • See $.access()

Announcing Dynamic Updates

Live Regions

  • Live regions are easy: div(aria-live="assertive|polite")
  • Update contents, screen reader speaks content

Live Region Caveats

  • Multiple live regions add complexity
  • Live region role support is unreliable (role="alert|log|status|timer")
  • Live regions can't be added dynamically to the DOM *

One live region to rule them all...

  • Use a single, global "announcer" live region
  • Provide common mechanism to "announce" content
  • Use a script - announce() - to make updates
Say anything w/announce()Check out $.announce() and a11y_kit on Github

With great power...

Use this technique judiciously

  • Rule #1: Follow core accessibility principles, best practices
  • Use announce() to communicate pertinent info only
  • Keep announced messages succinct

With great power... (continued)

Use this technique judiciously

  • Use cases:
    • View updates in single-page apps
    • Form validation errors
    • Alert messages (non-modal)
    • Progress updates in multi-step process

Modal Dialogs

Modals

Goals

  • User is aware of modal presence and purpose
  • User is able to navigate modal content
  • User is unable to navigate background content

Modals

Setup

  • H1 is first heading
  • Use role="dialog" to assign role to modal container
  • Use aria-labelledby to label modal
  • Use aria-describedby to describe modal (optional)

Modals

On Open

  • UI Isolation - hide main content using aria-hidden="true"
  • Place focus on first focusable element
  • Esc key closes dialog (unless action required)

Modals

On Close

  • Remove UI isolation
  • Place focus back on modal trigger

Tooltips

Tooltips

Design Tips

  • Tooltips should contain succinct, non-interactive, informational content only
  • Tooltips with interactive content (e.g. links, buttons) should be a modal

Tooltips

Behaviors

  • On hover/focus, it's OK to display a tooltip if contents are read to user
  • E.g. button(role="tooltip" aria-label="Tooltip content")

Dynamic Table Filtering and Sorting

Goals

  • Communicate of table purpose and functionality
  • Communicate status changes, avoid confusion

Overview

  • Table Navigation
  • Table Structure
  • Column Sorting
  • Table Summary
Data table widget example

Table Navigation in VoiceOver

  • VO-R - hear table row contents from the location of the VoiceOver cursor to the end of the row
  • VO-C - hear the column header
  • VO-C-C - hear table column contents from the location of the VoiceOver cursor to the bottom of the column
  • Up/Down Arrow - move up or down in a column
  • VO-| - sort the column, position the VoiceOver cursor on the column
  • VO-Command-\ - move to the parent row of the current column

Table Structure

  • Provide a title for your table via the caption tag
  • Use thead and tbody correctly
  • Use th(scope="col") to define table column headings
  • Use th(scope="row") to define table row headings
  • Column/row heading content should be succint but understandable

Column Sorting

  • Use button element to control column sorting
  • Use aria-sort='ascending|descending' on the sorted th
  • Use aria-label to communicate visual icon meaning
  • Update the table status and announce state change
  • Manage focus - ensure that focus stays on active button

Table Summary

  • Provide a table status that is connected to the table via aria-describedby
    • "Showing page 2 of 5, sorted descending by First Name"
    • "Now sorting by Last Name ascending"
  • Update and announce() the new table state when filtering, pagination and/or sorting changes
